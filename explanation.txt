magicpin AI Challenge — Vera Merchant AI Assistant: Full Explanation Guide
================================================================================

Aapko is challenge mein magicpin ke merchant-AI assistant "Vera" ko re-build/optimize karna hai. Vera ek WhatsApp AI chatbot hai jo merchants ko unka business grow karne, Google Business Profile (GBP) improve karne, and customers ke questions ka answer dene mein help karta hai.

Chaliye is project, datasets aur folders ko detail mein samajhte hain taaki aapko clear ho jaye ki EXACTLY kya build karna hai.

1. ARCHITECTURE: THE 4-CONTEXT FRAMEWORK
================================================================================
Vera ka core philosophy hai ki koi bhi message tabhi effective hoga jab uske paas correct context ho. Aapka chatbot har message ko compose karne ke liye niche diye gaye 4 contexts ka use karega:

1. CategoryContext (Slow-changing vertical info):
   - Ye batata hai ki business kis type ka hai (e.g., Dentist, Salon, Restaurant, Gym, Pharmacy).
   - Isme specific rules hote hain jaise:
     - Voice/Tone: Dentists ke liye clinical tone (no hype, use "Dr."), salons ke liye warm & friendly.
     - Taboos: Dentists ke liye "cure" or "guaranteed" words strictly banned hain.
     - Offer Catalog: Standard packages, e.g., "Dental Cleaning @ ₹299" (is type ke offer packages generic "10% off" discounts se zyada reply rates late hain).
     - Peer Stats: Uss category ke city-wide benchmarks (avg rating, average CTR, etc.).
     - Research Digest: Latest updates, e.g., JIDA trial details or new regulations.

2. MerchantContext (Specific business state):
   - Ye specific shop/business ki snapshot hoti hai.
   - Isme identity (business name, locality, preferred languages), subscription status (Active/Expired/Trial), past 30 days performance (views, calls, CTR), and active offers hote hain.
   - Isme signals bhi hote hain (jaise "stale_posts:22d" ya "ctr_below_peer_median") jisse pata chalta hai ki business mein kya problem chal rahi hai.

3. TriggerContext (The event/reason to message right now):
   - Ye batata hai ki message ABHI kyu jaa raha hai (urgency level 1-5).
   - Triggers do tarah ke hote hain:
     - External: Festive events (e.g., Diwali in 4 days), weather alerts (e.g., heatwave), category updates (e.g., new research paper released).
     - Internal: Performance dips (views dropped 40%), milestones reached (100 reviews reached), customer renewal reminders, or customer recall window opening.

4. CustomerContext (Optional - only when messaging a merchant's customer):
   - Jab Vera merchant ke behalf par customer se baat karti hai (e.g., sending appointment reminders or recall messages).
   - Isme customer's name, visit relationship (visits count, last visit date), preferences (prefers weekday evenings), and language pref (Hinglish/English) hota hai.

3. WHAT YOU HAVE TO BUILD (Aapko kya build karna hai?)
================================================================================

Aapko ek Python program/server build karna hai jo do major components ko handle karega:

Component A: HTTP API endpoints server (bot.py)
--------------------------------------------------
Aapko ek FastAPI ya Flask server banana hai jo local port (jaise 8080) par run karega. Isme 5 endpoints hone chahiye:

1. `GET /v1/healthz`
   - Healthcheck probe. Returns status "ok", uptime, aur kitne contexts load hue hain.
2. `GET /v1/metadata`
   - Details of your team: team_name, members, model used, approach.
3. `POST /v1/context`
   - Judge aapko category/merchant/customer/trigger data update karne ke liye isko hit karega. Aapko ye variables in-memory state mein store/update karne hain (idempotent design based on version).
4. `POST /v1/tick`
   - Clock advancement event. Judge aapko batayega ki current simulated time kya hai aur kaunse triggers active hain.
   - Aapke bot ko check karna hai ki kya koi trigger action laye-layak hai, agar hai to compose process run karke message payload prepare karega aur array of actions return karega.
5. `POST /v1/reply`
   - Jab merchant/customer aapke send kiye message ka reply karta hai, tab ye endpoint trigger hota hai.
   - Aapke bot ko reply return karna hai with either:
     - `action: "send"` (next reply body details)
     - `action: "wait"` (kuch time hold karne ke liye)
     - `action: "end"` (conversation complete ya unsubscribe exit karne ke liye)

Component B: The AI Composition Engine (LLM prompts)
--------------------------------------------------
Aapko LLMs (jaise GPT, Claude, ya Gemini) ka use karke ek prompt template design karna hai:
- Wo category (voice/taboos), merchant (performance issues/identity), trigger (Diwali, dip in calls), and optional customer details read karega.
- Wo ek unique/highly-compelling output message design karega.
- Output requirements:
  - Concise aur precise hona chahiye (no generic "I hope you are doing well").
  - Target values bilkul specific hone chahiye, e.g., "aapka CTR abhi 2.1% hai jo peer average 3.0% se kam hai."
  - Single binary CTA hona chahiye (Yes/No structure like: "Kya main iska abstract send karu?").
  - Local languages mix support (English mixed with Hindi like "Vera se bol rahi hoon, aapka Google profile 62% update ho gaya hai").

Component C: Conversation State Management
--------------------------------------------------
Kuch special cases handle karne hain:
1. Auto-Reply Detection: Agar merchant ka auto-reply/bot message continuous repeat ho raha hai (verbatim match 3+ times), toh hamare bot ko turn waste kiye bina directly gracefully conversation `end` kar dena chahiye.
2. Intent Handoff: Agar merchant ka response simple "Haan let's do it" or "ok start karo" hai, toh hume pitches se redirect hoke direct action execution state me jaana hai (not repeat basic questions).
3. Graceful Exit: Agar user rude reply kare ya "stop/unsubscribe" bole, toh sorry bolkar immediately transaction end karna hai.

4. EVALUATION & SCORING METHODOLOGY
================================================================================
Aapke chatbot ko Judge LLM 5 parameters (har parameter 0-10, total 50 marks) par score karega:

1. Specificity (10 Marks): Message me raw data or numbers hone chahiye, not generic marketing text.
2. Category Fit (10 Marks): Dentists ke liye doctor jaisa professional voice and salons ke liye friendly voice follow hona chahiye.
3. Merchant Fit (10 Marks): Merchant's local locality details and performance metrics match karne chahiye, and language (English/Hindi mix) natural hona chahiye.
4. Trigger Relevance (10 Marks): Message clearly hook hona chahiye uss event/reason se jisne trigger kiya hai.
5. Engagement Compulsion (10 Marks): Content me direct Call-To-Action (CTA) hona chahiye jisse merchant respond karne ke liye curious ho.

Penalties:
- Hallucinated details (agar context me data nahi hai aur aapne message me likh diya): -2 marks.
- Exposing system internal names/jargon: -1 marks.
- Verbatim duplicate messages: -2 marks.

5. NEXT STEPS (Aapko ab aage kya karna hai?)
================================================================================
1. **Initialize the bot server skeleton**: FastAPI build karke health endpoints check karein.
2. **Setup context database**: Memory dictionaries deploy karein category, merchant, customer aur triggers cache capture karne ke liye.
3. **Write prompt handler**: System templates configure karein parsing of 4-contexts.
4. **Deploy LLM API keys**: OpenAI/Gemini connectivity verify karein.
5. **Run Judge Simulator**: `python judge_simulator.py` command run karke continuous scoring test karein.

Aapka poora goal is process ko automated, responsive (under 30s response constraint) aur category fit banana hai.
